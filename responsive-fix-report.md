# Responsive fix report — header/hero/ticker

## Что было сломано
- Бургер отображался в диапазоне 900–1099px, но клик не открывал меню.
- На ширинах ≤899px шапка/меню выглядели «без стилей» и ломали расположение.
- Полоска с бегущей строкой внизу первого блока скакала/ломала высоту и могла накладываться на контент.

## Корневые причины
- Несовпадение брейкпоинтов CSS и поведения меню: мобильное меню было ограничено max-width: 899px, а бургер виден до 1099px.
- Ключевые стили кнопки/бургера были зашиты в min-width: 900px, из‑за чего на ≤899px элементы теряли базовый стиль.
- Высота полосы и тикера была задана с разными значениями в разных блоках, что приводило к перепадам высоты и смещению.

## Что изменено
- Уточнена логика `initHeader()` и синхронизация ARIA/видимости: бургер реагирует только когда реально виден, меню закрывается при переходе в desktop.
- Добавлен единый мобильный блок стилей для меню/бургера/CTA на ≤1099px, плюс мелкие корректировки для ≤520px и ≤390px.
- Зафиксированы канонические размеры тикера через `--stripe-ticker-h`, закрепление и высота тикера, а также нижний отступ у hero.
- Визуальные скриншоты: базовые и после фикса — сохранены в `tests/visual/baseline/` и `tests/visual/after/`.

## Файлы

## Как прогнать визуальные проверки
1) Запустить локальный сервер:

```
python3 -m http.server 8088
```

2) Снять базовые скриншоты:

```
OUTPUT_DIR=tests/visual/baseline npx playwright test tests/visual/visual.spec.js
```

3) Снять скриншоты после правок:

```
OUTPUT_DIR=tests/visual/after npx playwright test tests/visual/visual.spec.js
```
